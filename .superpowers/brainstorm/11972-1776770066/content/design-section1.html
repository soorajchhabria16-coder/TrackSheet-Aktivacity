Section 1 of 5 — Navigation & Role Views

Each role sees a different version of the app after logging in. Does this look right?

ADMIN
FULL ACCESS
  • ● Dashboard
  • ● Production Tasks (all)
  • ● Team
  • ● Admin Panel ★
  • ● Notifications
  • ● Settings
Admin Panel unlocks:
Invite users, change roles, remove users
PROJECT MANAGER
TASK CONTROL
  • ● Dashboard
  • ● Production Tasks (all)
  • ● Team (view only)
  • ● Admin Panel (hidden)
  • ● Notifications
  • ● Settings
Can do:
Create tasks, assign to team, update any task, view all reports
TEAM MEMBER
OWN TASKS ONLY
  • ● Dashboard (hidden)
  • ● My Tasks (assigned only)
  • ● Team (hidden)
  • ● Admin Panel (hidden)
  • ● Notifications
  • ● Settings
Can do:
View assigned tasks, change status (To Do → In Progress → Review → Done), leave comments
On login: The app reads the user's role from Supabase and shows only the pages and data that role is allowed to access. Team Members who try to access Admin or Dashboard directly are redirected back.
