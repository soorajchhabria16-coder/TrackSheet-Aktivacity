Section 2 of 5 — Database Changes

Three small changes to the Supabase tables to support roles, comments, and notifications.

profiles UPDATE
existing columns...
+ role
text DEFAULT 'member'
('admin' | 'pm' | 'member')
One new column. Every user gets a role. Admins can change it from the Admin Panel.
comments NEW
id uuid PK
task_id uuid → tasks
user_id uuid → profiles
content text NOT NULL
created_at timestamptz
Stores all comments on tasks. Both PMs and Team Members can post. Read access scoped by role.
notifications NEW
id uuid PK
user_id uuid → profiles
task_id uuid → tasks
type text
('assigned' | 'status_change' | 'comment')
message text
read bool DEFAULT false
created_at timestamptz
One row per notification. The bell icon count = unread rows for the current user.
What triggers a notification:
  • Task assigned → Team Member gets "assigned" notification
  • Team Member changes status → PM gets "status_change" notification
  • Anyone comments on a task → the other party gets a "comment" notification
